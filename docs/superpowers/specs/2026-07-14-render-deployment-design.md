# Spec: Render Deployment (Ephemeral SQLite)

**Date:** 2026-07-14
**Topic:** Deploying RainSense Flask App to Render

## Goal
Deploy the RainSense Flask-based weather classification application onto Render using Render's native Python buildpack and an ephemeral SQLite database for demo purposes.

## Design Decisions
- **Hosting Platform:** Render (Web Service)
- **Environment:** Native Python 3.12 Buildpack
- **WSGI Server:** Gunicorn (to run `run:app` in production)
- **Database:** Ephemeral SQLite. The sqlite database will reside in the container (`weather.db`) and initialize on startup. 
  - *Note:* User registrations and prediction history will reset whenever the Render Web Service restarts or scales down due to inactivity. This is acceptable for demonstration/testing.
- **Environment Variables:**
  - `PYTHON_VERSION`: `3.12.3`
  - `SECRET_KEY`: Automatically generated by Render
  - `DATABASE_PATH`: `./weather.db`

## Proposed Changes

### Requirements Update
Add `gunicorn` to [requirements.txt](file:///c:/Users/wisdo/OneDrive/Documents/GitHub/weather_classification_model/requirements.txt) to serve the Flask app in production.

### Render configuration
Create [render.yaml](file:///c:/Users/wisdo/OneDrive/Documents/GitHub/weather_classification_model/render.yaml) in the root directory to define the Render Web Service configuration.

```yaml
services:
  - type: web
    name: rain-sense
    env: python
    buildCommand: pip install -r requirements.txt
    startCommand: gunicorn run:app
    envVars:
      - key: PYTHON_VERSION
        value: 3.12.3
      - key: SECRET_KEY
        generateValue: true
      - key: DATABASE_PATH
        value: ./weather.db
```
